What it is

A data quality agent that profiles a dataset, asks Claude to propose a focused set of typed rules (not_null, unique, min/max, in_set, regex, max_null_fraction) via tool-use, then executes those rules in plain pandas and reports violations. It exits non-zero on failure so it drops straight into a CI or Airflow gate.

Design & architecture

The model never declares data "good" — it only emits typed rules. Enforcement is plain code you can read in a diff.

How I built it

  • Compact profile: per-column dtype, null fraction, cardinality, min/max/mean or top values, plus a few sample rows.
  • Tool-use schema: Claude must return rules as structured JSON — no fragile text parsing.
  • Deterministic execution: each rule type maps to an exact pandas check; a hallucinated rule is a visible, reviewable artifact, never silent data corruption.
  • CI-friendly: non-zero exit on any failure.
