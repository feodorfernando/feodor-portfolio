🚀 Next-Gen Databricks Platform

One codebase that runs in dev / test / prod and routes itself to the right catalog, secrets from Azure Key Vault, streaming and batch ingestion through to gold, CI/CD via Asset Bundles — and Spark jobs that survive OOM, skew and transient errors by re-tuning configuration at runtime.

What it is

An industry-standard medallion platform built the way mature teams actually ship Databricks: environment isolation through Unity Catalog, secrets in Key Vault, both ingestion modes, clusters defined as code, automated promotion dev → test → prod, and resilient Spark that doesn't fall over the first time a partition is too big. It directly addresses the operational realities — "how do I run the same notebook in dev and prod without editing it?", "where do credentials live?", "what happens when a job OOMs at 2am?" — that separate a demo from a platform.

Design & architecture

Dynamic, environment-aware code

The same notebook runs everywhere. config/environments.py resolves the environment (job param → cluster env var → workspace URL → safe default of dev) and returns a config that routes every read/write. No hard-coded catalog names; no "oops, wrote to prod".

cfg = get_config(spark, dbutils)          # dev / test / prod, auto-detected
silver = cfg.table(cfg.silver_schema, "events")
#   dev  workspace -> dev_catalog.silver.events
#   prod workspace -> prod_catalog.silver.events
assert_safe_to_write(cfg)                 # prod writes gated behind ALLOW_PROD_WRITES

Handling OOM & runtime errors — dynamically

Prevention plus recovery. The session is tuned AQE-first to avoid most OOM/skew; if a stage still fails, an adaptive_run decorator classifies the failure and changes the configuration at runtime, then retries — rather than dying.

@adaptive_run(spark, max_attempts=3)   # OOM → ↑shuffle partitions, ↓split size, broadcast off
def merge_silver():                    # skew → strengthen AQE skew join, ↑partitions
    ...                                # transient → backoff & retry
merge_silver()                         # unknown → raise (don't mask real bugs)

Bad records (vs infra failures) are quarantined to a dead-letter table instead of being dropped silently. AQE skew-join handling is on by default, with a manual salting helper for pathological single-key skew.

Ingestion: streaming + batch

  • Streaming — Auto Loader → bronze with schema evolution, checkpointed exactly-once, and maxBytesPerTrigger to bound per-batch memory (anti-OOM).
  • Batch — idempotent COPY INTO (re-runs skip already-loaded files), env-aware target table.
  • Silver/Gold — MERGE upsert with dead-lettering; gold aggregates annotated with business-language comments so the BI Copilot / Genie answer accurately.

Secrets, clusters & CI/CD

  • Azure Key Vault → Databricks secret scopes (kv-dev / kv-prod); credentials never live in code. UC storage credentials preferred where available.
  • Clusters as code — autoscaling job clusters, Photon, spot-with-fallback, AQE on, defined in the Asset Bundle.
  • CI/CD — databricks bundle deploy via GitHub Actions: PR → dev, main → test, tagged release → prod. Prod writes gated behind an explicit flag.
