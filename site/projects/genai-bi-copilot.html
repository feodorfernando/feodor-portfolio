What it is

A self-service analytics copilot for the BI team. Analysts queue behind the data team for ad-hoc questions; this removes the queue for the large fraction of questions answerable from certified gold tables. Claude is grounded only on the certified gold schema and metric definitions, generates a query, and every query passes a safety guard (single statement, read-only, gold-only, LIMIT-enforced) before it executes. It returns the result, a plain-English summary, and the exact SQL it ran so analysts can verify and reuse it.

Design & architecture

How I built it

  • Grounded generation — Claude only sees the certified gold schema + metric definitions (built from Unity Catalog comments / a metrics manifest), so it doesn't invent tables or redefine "revenue".
  • Tool-use for SQL — the model emits SQL via a structured tool call, not free text.
  • Safety guard — single statement, SELECT/WITH only, every table inside the allow-listed gold schema, LIMIT injected. A copilot must be unable to mutate data.
  • Narrate the result — a cheap/fast model (Haiku) turns the result set into a 2–3 sentence business insight; the exact SQL is returned for transparency.
