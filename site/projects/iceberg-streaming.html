What it is

A working streaming pipeline that lands a Kafka event stream into Apache Iceberg. Raw events are appended to a bronze table; a deduplicated, late-data-corrected copy is upserted into a silver table via idempotent MERGE using merge-on-read so each micro-batch commit stays cheap. It ships with the maintenance jobs (compaction, snapshot expiry, orphan cleanup) that make streaming Iceberg viable in production — and a deep research note on Iceberg internals.

Design & architecture

How I built it

  • Bronze/silver split — append raw cheaply, then MERGE deduped rows so corrections and replays are idempotent.
  • Merge-on-read tables (write.*.mode = merge-on-read) so per-micro-batch upserts write small delete files instead of rewriting whole files.
  • Late/out-of-order handling via MERGE … WHEN MATCHED AND s.event_time > t.event_time.
  • Exactly-once-ish from Spark checkpoints + idempotent MERGE; failed batches leave orphans that maintenance cleans.
  • Maintenance job runs compaction, position-delete rewrite, snapshot expiry, and orphan removal out-of-band so it never blocks ingestion.
