🔄 Real-Time CDC → Iceberg

Mirror an operational database (Postgres/MySQL) into Apache Iceberg in near-real-time via Debezium CDC — applying inserts, updates, and deletes as row-level operations with idempotent, LSN-guarded MERGEs.

What it is

A streaming consumer that turns a database's own change log into a continuously-updated Iceberg replica. Debezium publishes change events (insert/update/delete) to Kafka; the job parses the Debezium envelope, collapses to the latest change per primary key per micro-batch, and applies a MERGE that upserts non-deletes and applies tombstone deletes — guarded by the source LSN so replay and out-of-order events are idempotent.

Design & architecture

How I built it

  • Debezium envelope parsing (before/after/ op/lsn/ts_ms) into a generic row image, so it works for any source table without a hard-coded schema.
  • Latest-per-key collapse within each batch by LSN.
  • LSN-guarded MERGE: only apply a change newer than what's stored → idempotent under replay and out-of-order delivery.
  • Snapshot → stream handoff: read from earliest so the Debezium initial snapshot seeds the table, then stream changes — identical handling for both.
  • Merge-on-read so deletes/updates are cheap; pairs with the maintenance job from the Iceberg Streaming project (CDC accumulates delete files fast).
