What it is

An end-to-end Azure data pipeline modeled on a real enterprise analytics platform. Streaming telemetry flows through Event Hub → Stream Analytics for sub-minute dashboards; operational stores (Azure SQL, Cosmos DB, Dataverse) are incrementally copied by Azure Data Factory into ADLS gen2, transformed in Databricks (medallion), and served through Synapse serverless to Power BI. It includes the actual Stream Analytics queries, ADF pipeline definition, and Synapse external-table SQL.

Design & architecture

How I built it

  • Latency-driven ingestion split: streaming telemetry via Stream Analytics (sub-minute); slowly-changing operational tables via ADF incremental copy on a modified_at watermark — one tool per job.
  • Hot path: Event Hub → Stream Analytics tumbling-window aggregates → Power BI live tiles, plus an anomaly path to a Service Bus alert queue.
  • Cold path: ADF copy → ADLS gen2 bronze → ADF triggers a Databricks medallion transform → gold Parquet.
  • Serverless serving: Synapse serverless SQL external tables/views over gold — no idle dedicated pool cost; Power BI queries the views.
